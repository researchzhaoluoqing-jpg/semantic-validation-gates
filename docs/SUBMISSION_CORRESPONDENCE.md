Framework for Semantic Verification*, SSRN Abstract ID 7031058) was declined
at screening. SSRN states it does not reconsider rejections and gives no
individualized reasons, so **no appeal should be filed**. The v3.2 manuscript

Both drafts below require the author's review before sending. In particular,
**verify the AI-use description against your own drafting history for v3.1**

routed to the "appeal" bin), states the differences as verifiable facts
rather than as arguments about merit, and explicitly invites a scope answer —
which is the single most useful thing they can tell you.

## C. If you prefer the recommended primary route instead

Emailing SSRN is low-cost but also low-expected-value: at best they confirm
the submission route, at worst they confirm the scope is wrong. The higher-
value action is unchanged:

- **TMLR** (OpenReview, rolling submissions, no page limit) — gives peer
  review, and the released code + pre-registered protocol are strengths there
  rather than merely nice extras.
- **Zenodo** — immediate citable DOI, no gatekeeping, establishes the
  timestamp while TMLR review runs.

These are not mutually exclusive with a SSRN submission, but if effort is
scarce, spend it on TMLR formatting rather than on SSRN correspondence.
